Architecture Details
Data Flow: Collector (API fetch → CSV cache) → Engine (CSV → indicators in SQLite) → Dashboard (SQLite → browser)
Parallelism: Collection uses ThreadPoolExecutor (I/O-bound), computation uses ProcessPoolExecutor (CPU-bound). Multiple watchlists can run concurrently.
Market Calendar: NYSE holidays computed algorithmically. DST-aware (US Eastern). After-close mode triggers post-16:00 ET on trading days only.
Indicators: MOM, SQZ, BAND, ACC, DEV, 30WMA across 14 timeframes (1H–12M).
Storage: SQLite (WAL mode) for indicators and events. CSV cache for raw OHLC data.

📡 Process 1 — Data Collector

Fetches OHLC price data from the data source and caches to local CSV files.

Data Sourceyfinance (Yahoo Finance) Price AdjustmentSplit-adjusted always, dividend toggle (ADJ) Trading HoursExtended (pre + post market included) WatchlistsSP500 (473 tickers), Priority, custom CacheLocal CSV files per ticker (incremental updates)

⚙ Process 2 — Computation Engine

Reads cached CSV data and computes all indicators across all timeframes. Writes results to SQLite.

IndicatorsMOM, SQZ, BAND, ACC, DEV, 30WMA Timeframes1H, 4H, 8H, 1D, 2D, 3D, 5D, 1W, 2W, 6W, 1M, 3M, 6M, 12M WorkersProcessPoolExecutor (CPU-bound) Change DetectionTracks indicator state changes and logs events StorageSQLite (momentum_dashboard.db)

🌐 Process 3 — Dashboard Server

Serves the web interface and API. Reads from SQLite to render current state.
